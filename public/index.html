Star Wars Chatbot Eval API

Test endpoint for evaluating a Star Wars chatbot with Arize remote evaluators, Agent-as-a-Judge, and custom skills. May the Force be with your evals.

Vercel Serverless TypeScript Remote Evaluator Agent-as-a-Judge Star Wars Canon

Endpoints

POST /evaluate Arize remote-evaluator contract — scores chatbot output, returns label, score, explanation
POST /tool/score Scoring tool for Agent-as-a-Judge custom skills — accepts text + criteria
GET /health Warmup / config status (unauthenticated)
GET /requests Debug buffer — last 50 captured payloads, including what Arize sent (auth required)
DEL /requests Clear the debug buffer between test runs (auth required)

Eval Criteria

Pass criteria via ?criteria=, x-eval-criteria header, or embed in the evaluator name: llm-hallucination.

lore_accuracy
Lore Accuracy
All stated facts match Star Wars canon (films, TCW, Rebels)
hallucination
Hallucination
Response does not invent non-canon characters, events, or planets
relevance
Relevance
Response directly addresses the user's Star Wars question
in_character
In Character
Response maintains the bot's Star Wars persona (e.g. Yoda syntax)

Test Modes

Set via ?mode= query, x-test-mode header, or the Arize evaluator name prefix. Name your evaluator stub-pass and it always returns pass — no headers needed.

keyword (default)
Regex scan for Jedi, Sith, Force, etc. in the output
stub-pass
Always pass / 1 — baseline happy-path test
stub-fail
Always fail / 0 — verify Arize records failures
llm
Real model call with Star Wars canon judge prompt + selected criteria
no-verdict
200 with empty body — tests Arize's "no label/score" failure path
slow-<ms>
Delayed response — cold-start & timeout testing
force-429
Rate-limit — Arize backs off and retries
force-500
Transient error — Arize retries
force-401 / force-403
Fatal auth — Arize stops retrying entirely
force-400
Record failure — no retry

Example — /evaluate

# What Arize sends to POST /evaluate { "metadata": { "request_id": "uuid-for-this-call", "evaluator": "lore_accuracy", "record_id": "span-abc123" }, "input": { "input": "Who trained Anakin Skywalker?", "output": "Obi-Wan Kenobi was Anakin Skywalker's Jedi Master." } } # Response (keyword mode — detects "Jedi") { "label": "pass", "score": 1, "explanation": "Output references Star Wars lore (Jedi, Sith, the Force, etc.)." }

Example — /tool/score

# Called by the Agent-as-a-Judge custom skill POST /tool/score { "record_id": "span-abc123", "criteria": "hallucination", "text": "Qui-Gon Jinn discovered midi-chlorians and trained Obi-Wan Kenobi." } # Response { "label": "pass", "score": 1.0, "rationale": "Answer does not invent non-canon Star Wars facts.", "criteria": "hallucination", "source": "arize-test-eval-api/tool/score" }

Auth

Set EVAL_AUTH_TOKEN in Vercel environment variables. Add Authorization: Bearer <token> to protected endpoints. Configure the same header on the Arize remote evaluator. /health is always open for warmup pings.
