Arize Test Eval API

Test endpoint for Arize remote evaluators, Agent-as-a-Judge, and skills.

Vercel Serverless TypeScript Arize Remote Evaluator Agent-as-a-Judge

Endpoints

POST /evaluate Arize remote-evaluator contract — returns label, score, explanation
POST /tool/score Scoring tool for Agent-as-a-Judge custom skills
GET /health Warmup / config status (unauthenticated)
GET /requests Debug buffer — last 50 captured payloads (auth required)
DEL /requests Clear the debug buffer (auth required)

Test Modes

Set via ?mode= query param, x-test-mode header, or the Arize evaluator name prefix (e.g. name your evaluator stub-pass).

keyword (default)
Rule-based regex scoring on output field
stub-pass
Always returns pass / 1
stub-fail
Always returns fail / 0
llm
Real OpenAI or Anthropic call, parsed verdict
no-verdict
200 with empty body — Arize marks as record failure
slow-<ms>
Delays response — tests timeouts & cold-start warmup
force-429
Rate-limit path — Arize backs off and retries
force-500
Transient error — Arize retries
force-401 / force-403
Fatal auth — Arize stops retrying
force-400
Record failure — no retry

Example Request

# Arize sends this to /evaluate { "metadata": { "request_id": "uuid-for-this-call", "evaluator": "stub-pass", "record_id": "span-abc123" }, "input": { "input": "Which restaurants deliver?", "output": "Olio e Piu delivers via DoorDash." } } # Response { "label": "pass", "score": 1, "explanation": "stub: always pass" }

Auth

Set EVAL_AUTH_TOKEN in Vercel environment variables. Protected endpoints require Authorization: Bearer <token>. Add the same header in the Arize evaluator config.
